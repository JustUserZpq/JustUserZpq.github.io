Java篇

Map和HashMap的区别

Map 和 HashMap 都是 Java 中用于存储键值对的数据结构。它们之间的主要区别在于：

  • 实现方式：Map 是一个接口，定义了一些方法来管理键值对。而 HashMap 则是 Map 接口的实现类之一，使用基于哈希算法实现。
  • 线程安全性：HashMap 不是线程安全的。如果多个线程同时访问 HashMap，可能会导致数据不一致。另一方面，Java 提供了一些线程安全的 Map 实现，例如 ConcurrentHashMap。
  • 插入顺序：HashMap 不保证插入顺序，即使在 JDK8 中 HashMap 也不是有序的。如果需要有序的键值对集合，可以使用 LinkedHashMap。
  • null 值：HashMap 允许将键和值设置为 null，而 Hashtable 则不允许。

简述 java 垃圾回收机制

Java中的垃圾回收机制是自动的内存管理系统，可以自动检测和释放程序运行过程中不再使用的内存空间，避免了手动内存管理带来的困难。其主要思想是通过垃圾回收器扫描堆内存中的对象，判断哪些对象没有被引用，然后将这些对象所占用的内存空间释放出来，以供其他对象使用。

Java中的垃圾回收机制主要由以下组件构成：

  1. 垃圾回收器（Garbage Collector）：负责监视程序中的对象，并在它们变成“垃圾”时将其回收释放内存空间。
  2. 引用计数器（Reference Counter）：记录每个对象被引用的次数，当对象的引用计数为0时，该对象被标记为垃圾并由垃圾回收器回收。
  3. 可达性分析算法（Reachability Analysis）：是Java中最广泛应用的垃圾收集算法，它通过从根集（如全局变量、静态变量等）开始遍历程序的对象图，标记出所有活跃的对象，并且在标记完成之后将未被标记的对象回收。
  4. 内存分代模型（Memory Generational Model）：将堆内存分为不同的代，并对不同代使用不同的垃圾回收算法和频率，以此提高垃圾回收的效率。
  5. 终结器（Finalizer）：在对象被垃圾回收前可以自动执行一些清理操作，例如关闭文件、释放系统资源等。

总体来说，Java中的垃圾回收机制使得程序员可以专注于业务逻辑而无需关注内存管理问题，提高了程序的开发效率和可靠性。但是，过度依赖垃圾回收机制也会导致内存泄漏和性能问题，所以在实际编程中仍需要注意内存管理的问题。

java中垃圾收集的方法有哪些?

  1. 标记-清除算法（Mark-and-Sweep）：该算法分为两个阶段。第一阶段，从根节点开始遍历对象图，在已经遍历的对象上打上标记；第二阶段，遍历整个堆空间，如果发现对象没有被标记，则将其释放掉，否则清除标记。这种方法会导致内存碎片问题。
  2. 复制算法（Copying）：该算法将堆空间划分为两个区域，每次只使用一个区域，在进行垃圾回收时，将存活的对象复制到另一个区域，然后将原区域中的所有对象一次性清除掉。这种方法不会出现内存碎片问题。
  3. 标记-整理算法（Mark-and-Compact）：该算法类似于标记-清除算法，但是在清除未标记对象的同时，还会将存活的对象往一端移动，然后清除边界以外的对象。这种方法也不会出现内存碎片问题。

在实际应用中，Java 虚拟机可以选择不同的垃圾回收算法和策略，以满足特定的需求。例如，早期版本的 Hotspot JVM 使用的是分代收集算法，将堆空间分为新生代和老年代，并分别使用不同的垃圾回收器来处理不同代的对象。而在较新的版本中，Hotspot JVM 还引入了 G1 垃圾回收器，它基于分代收集算法和复制算法，在避免内存碎片的同时提供了更好的吞吐量和延迟性能。

MySQL篇

索引语法

1). 创建索引

CREATE [ UNIQUE | FULLTEXT ] INDEX index_name ON table_name (index_col_name,… ) ;

2). 查看索引

SHOW INDEX FROM table_name ;

3). 删除索引

DROP INDEX index_name ON table_name ;

使用索引的优势

使用索引可以提高数据查询的效率

  1. 快速定位数据：索引可以帮助数据库快速定位需要查询的数据，避免了全表扫描的低效性能。通过创建最适合特定查询条件的索引，可以使数据库在大型数据集上处理查询时以更快的速度返回结果。
  2. 加速排序：当查询结果需要根据某个字段进行排序时，索引可以加快排序的速度，减少查询时间。
  3. 提高并发性能：索引能够降低锁的竞争，提高并发性能。如果没有索引，需要对整个表进行锁定，这会导致其他查询阻塞。
  4. 减少IO操作：索引可以减少磁盘IO操作，因为不再需要访问整个表来查找满足查询条件的记录。相反，只需要访问索引所指向的数据块即可。
  5. 优化查询计划：数据库通过索引来选择使用哪种查询计划，以优化查询性能。如果没有索引，查询计划可能不是最优的，导致查询效率低下。

总之，索引可以极大地提高数据库的查询效率和性能，尤其是在大型数据集和高并发场景下，索引的优势更加明显。但是，过多的索引也会对数据库性能产生负面影响，因为索引需要占用存储空间和额外的维护成本。所以，在创建索引时需要根据实际情况进行权衡和选择。
